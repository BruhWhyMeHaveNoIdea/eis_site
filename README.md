# Tender Intelligence: парсер и ML-анализ тендеров ЕИС

Интегрированное решение для мониторинга закупок с [Единой Информационной Системы (zakupki.gov.ru)](https://zakupki.gov.ru): асинхронный сбор данных за последний месяц, хранение в БД Django и ML-анализ по запросу пользователя.

## Архитектура

```text
  ┌──────────────────────────────────────────────────────────────────┐
  │                        main_site (Django)                        │
  │                                                                  │
  │   ┌──────────┐   ┌────────────────────────────────────────────┐  │
  │   │ SQLite   │   │                  API                       │  │
  │   │  БД      │◄──│  GET /parser/start    → запуск парсера    │  │
  │   │ Tenders  │   │  GET /api/analyze/id → запуск ML-анализа  │  │
  │   └────┬─────┘   └────────────────────────────────────────────┘  │
  │        │                                                         │
  └────────┼─────────────────────────────────────────────────────────┘
           │
    ┌──────┴──────┐          ┌─────────────────────────────────────┐
    │ eis_parser  │          │               eis_ml                │
    │ (Scrapling) │          │  preprocessing → feature engineering│
    │             │          │  → HDBSCAN → similarity learning   │
    │ LinkSpider  │          │  → FAISS index                     │
    │ TenderSpider│          │  → прогноз цены (weighted median)  │
    └─────────────┘          └─────────────────────────────────────┘
```

**Поток данных:**

1. **Сбор** — `eis_parser` парсит тендеры с zakupki.gov.ru за последний месяц и сохраняет их в БД Django (модель `Tenders`).
2. **Просмотр** — пользователь заходит на главную страницу, видит таблицу тендеров, применяет фильтры (регион, способ закупки, этап, цена, дата, текстовый поиск).
3. **ML-анализ** — по запросу пользователя Django вызывает `eis_ml/analyze_tender.py` через subprocess: загружает предобученную модель, находит топ-k похожих исторических тендеров, прогнозирует цену и возвращает JSON.

## Компоненты

### eis_parser — сбор данных

Асинхронный парсер на базе [Scrapling](https://github.com/D4Vinci/Scrapling). Обходит страницы поиска zakupki.gov.ru и извлекает полные карточки тендеров по 44-ФЗ.

**Режимы работы:**

| Режим | Точка входа | Результат |
|-------|-------------|-----------|
| CLI | `python main.py` | JSON в `art/tenders.json` |
| API | `GET /parser/start` | Данные импортируются в БД Django |

**Парсер работает в два этапа:**
1. `LinkSpider` — обходит страницы пагинации и собирает ссылки на карточки тендеров.
2. `TenderSpider` — асинхронно парсит каждую карточку, извлекая все секции (`section__title` / `section__info`), очищает текст через `simple_clean()`.

**CLI-запуск:**
```bash
python main.py <start_page> <end_page> <per_page> <pub_date> [--close-date <close_date>]
```

| Аргумент | Тип | Описание |
|----------|-----|----------|
| `start_page` | int | Начальная страница пагинации |
| `end_page` | int | Конечная страница пагинации |
| `per_page` | int | Записей на странице (10/50/100) |
| `pub_date` | str | Дата публикации `ДД.ММ.ГГГГ` |
| `--close-date` | str | Дата закрытия `ДД.ММ.ГГГГ` (опц.) |

> **Ограничение:** `(end_page - start_page) * per_page <= 1000`

**Django API** вызывает `parser/parser.py` через subprocess. Результат (JSON в stdout) обрабатывается `update_tenders_from_data()` — каждая запись маппится в модель `Tenders` и сохраняется через `update_or_create` (по ИКЗ).

### eis_ml — ML-аналитика

Модуль ретроспективного анализа тендеров на основе мультиязычных эмбеддингов (Sentence Transformers, `paraphrase-multilingual-MiniLM-L12-v2`) и неконтролируемого обучения. Не использует ключевые слова — только семантическое сходство.

**Пайплайн обучения:**
1. `core.preprocessing` — очистка и нормализация русскоязычных данных (числа, даты, кириллица).
2. `core.embeddings` — генерация текстовых эмбеддингов через Sentence Transformers.
3. `ml.feature_engineering` — извлечение признаков: текстовые эмбеддинги + категориальные + числовые + временные.
4. `ml.clustering` (HDBSCAN + UMAP) — плотностная кластеризация для обнаружения паттернов.
5. `ml.similarity_learning` — контрастивное дообучение эмбеддингов под специфику закупок.
6. `ml.faiss_index` — индексирование для быстрого поиска ближайших соседей.

**Два алгоритма поиска похожих:**

| Алгоритм | Файл | Подход | Когда использовать |
|----------|------|--------|-------------------|
| **SelectKBest** | `select_k_best.py` | Правило-взвешенное сходство (косинус + категории + цена + дата) | До 10 тыс. тендеров, без обучения, высокая интерпретируемость |
| **MLRetrospective** | `ml_retrospective.py` | Кластеризация + контрастивное обучение + FAISS | 100 тыс.+ тендеров, максимальная скорость (O(log n)) |

**Прогнозирование цены:**
- `SimplePricePredictor` — взвешенная медиана цен топ-k похожих тендеров (методы: `mean`, `median`, `weighted_median`, `min`, `max`).

**Основные скрипты:**

| Скрипт | Назначение |
|--------|-----------|
| `train_ml_model.py -d <data.json>` | Обучение модели → `models/ml_retrospective_model.pkl` |
| `analyze_tender.py <tender.json> --json` | Анализ тендера: похожие + прогноз цены (вызывается Django API) |
| `select_k_best.py --input ... --target ...` | Поиск похожих без обучения |
| `compare_algorithms.py` | Сравнение двух алгоритмов на тестовой выборке |
| `price_predictor_simple.py` | Прогноз цены на основе похожих тендеров |

### main_site — веб-интерфейс

Django 6.x + Django REST Framework + SQLite.

**Модель `Tenders`** — полная карточка тендера 44-ФЗ:
- ИКЗ (уникальный ключ), наименование, заказчик, регион
- Способ закупки, этап, даты (начало/конец подачи, итоги)
- НМЦК, требования к участникам, обеспечение заявки/контракта
- Контакты (email, телефон), ЭТП

**API-эндпоинты:**

| URL | Метод | Описание |
|-----|-------|----------|
| `/` | GET | Главная страница: таблица тендеров с фильтрами |
| `/parser/start` | GET | Запуск парсера → данные в БД |
| `/api/analyze/<id>/` | GET | ML-анализ тендера: похожие + прогноз цены |

**Фильтры на главной странице:** регион, способ закупки, этап, диапазон дат окончания, диапазон цен, текстовый поиск (по наименованию, заказчику, ИКЗ).

**ML-анализ (`/api/analyze/<id>/`):**
1. Django извлекает тендер из БД, формирует JSON и сохраняет во временный файл.
2. Запускает `eis_ml/analyze_tender.py` через subprocess с флагом `--json`.
3. `analyze_tender.py` загружает предобученную модель, находит похожие тендеры, прогнозирует цену.
4. Результат возвращается как JSON response.

**Настройка путей** (`main_site/core/settings.py`):
- `EIS_PARSER_ROOT` — корень репозитория (где `parser/`)
- `EIS_ML_ROOT` — путь к `eis_ml/`

## Быстрый старт

### 1. Клонирование
```bash
git clone https://gitverse.ru/Vprs/eis_parser.git
cd eis_parser
```

### 2. Единое виртуальное окружение
Все зависимости (парсер, Django, ML) собраны в одном `requirements.txt`:
```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

> **Примечание:** `parser/requirements.txt` и `eis_ml/requirements.txt` сохранены для справки, но развёртывание идёт из корневого файла.

### 3. Обучение ML-модели
```bash
cd eis_ml
python train_ml_model.py -d tenders.json
```

### 4. Запуск веб-интерфейса
```bash
cd main_site
python manage.py migrate
python manage.py runserver
```

### 5. Использование
1. Откройте `http://127.0.0.1:8000/` — таблица тендеров.
2. Нажмите «Запуск парсера» (`/parser/start`) — данные за последний месяц сохранятся в БД.
3. Выберите тендер и нажмите «Анализ» — ML найдёт похожие и спрогнозирует цену.
