# Парсер для Единой информационной системы
### Описание проекта
В начале второго семестра ввиду введения в нашу образоваетльную программу Проектного практикума, наша команды встала перед сложным и судьбоносным выбором проекта. Проведя мозговой штурм внутри команды мы остановились на интересном, на наш взгляд проекте - «Neuromus Tender Intelligence: мониторинг ЕИС, отбор релевантных закупок и прогноз оптимального снижения».
Тогда мы еще не подозревали, к чему нас это приведет, но об этом позже.
Сразу обозначим **цель** нашего проекта:
>Создать на платформе Нейромус ИИ-решение для тендерного отдела, которое ежедневно:
>- Находит и отбирает закупки на ЕИС, точно соответствующие направлениям деятельности компании и заданным фильтрам (заказчик/регион/виды работ), формируя готовую выборку тендеров для проработки сотрудником.
>- Автоматически формирует аналитику по выигранным/проигранным закупкам на основе открытых данных (ЕИС + ФНС + площадки), включая список потенциальных участников и возможные взаимосвязи.
>- Для выбранной закупки строит прогноз вероятного снижения цены конкурентами и предлагает вариативность нашего снижения с учётом статистики конкурентов и опыта компании.

Вам конечно стало интересно, что такое Neuromus. Ваше любопытство похвально, однако я и сам не знаю, но это сейчам не так важно. Сосредоточимся на цели.

Одной из задач, которая стояла на пути к достижению нашей цели проекта, была задача автоматизированного сбора данных с [Единой Информационнай системы в сфере закупок](https://zakupki.gov.ru/epz/main/public/home.html) (далее ЕИС).
Ввиду отстутствия подходящего нам и бесплатного API, пришлось оперативно решать вопрос об автоматизации сбора данных. 

К счастью, у меня уже имелся опыт в написании автоматизированных скриптов для парсинга (о законности данного метода мы конечно лучше умолчим)
Ранее я, конечно писал парсеры, но они подходили только для отложенного сбора данных, ввиду своей медленной работы, т.к. использовался Selenium с UndetectedWebdriver для обхода ограничений.
Поэтому встала задача найти концептуально новый подход, который бы позволил парсить огромные кластеры данных в разумные строки

И эта задача, как вы понимаете, была решена: **[Scrapling](https://github.com/D4Vinci/Scrapling)**
Потенциально новая и свежая библиотека позволила выстроить весьма простой и эффективный пайплайн, благодаря ассинхронным веб-скраперам
Ну а код решения вы и сами можете увидеть в данном репозитории

### Интструкция по запуску
1. Клонируем репозиторийr себе на рабочую лошадку:
```
git clone https://gitverse.ru/Vprs/eis_parser.git
```
2. Создаем и активируем для него виртуальное окружение удобным способом, например:
```
python -m venv venv
source .venv/Scripts/activate
```
3. Устанавливаем все необходимые зависимости:
```
pip install -r requirements.txt
```
4. Создаем объект класса Parser(start_page, end_page, per_page, pub_date, close_date) с аргументами:
    - start_page - стартовая страница в пагинации (с какой начинаем)
    - end_page - конечная страница (какой заканчиваем). Должна быть >= start_page
    - per_page - сколько объектов выводится на одну страницу в пагинации на сайте (рекомендуются значения: 10, 50, 100). Не забываем, что с учетом страниц, не получится спарсить более 1000 записей
    - pub_ date - дата начала публикации тендера (Формат: ДД.ММ.ГГГ)
    - close_date - ОПЦИОНАЛЬНО дата закрытия тендера (Формат: ДД.ММ.ГГГ)
