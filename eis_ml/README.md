## Обзор проекта

Этот проект предоставляет два взаимодополняющих алгоритма для ретроспективного анализа государственных закупочных тендеров. Алгоритмы находят похожие исторические тендеры с использованием эмбеддингов (без сопоставления по ключевым словам).

### Ключевые принципы
- **Только эмбеддинги**: Использует мультиязычные модели Sentence Transformers
- **Поддержка русского языка**: Обрабатывает кириллический текст, форматы чисел и дат
- **Легковесность**: Умеренные требования к ресурсам
- **Готовность к интеграции**: Совместим с выводом парсера в формате JSON

### Два алгоритма
1. **Алгоритм 1: Simple select_k_best** - Правило-взвешенное сходство для наборов данных среднего размера (до 10 тыс. тендеров)
2. **Алгоритм 2: ML-based approach** - Неконтролируемое обучение с кластеризацией для больших наборов данных (100 тыс.+ тендеров)

## Алгоритм 1: Simple select_k_best

### Обзор
Правило-взвешенная оценка сходства, которая находит k наиболее похожих исторических тендеров с использованием эмбеддингов и специализированных метрик для данных о закупках на русском языке.

### Архитектура
- `preprocessing.py` - Очистка и нормализация данных
- `embeddings.py` - Генерация текстовых эмбеддингов
- `similarity.py` - Функции расчета сходства
- `select_k_best.py` - Основной алгоритм
- `config.py` - Конфигурация и веса
- `test_select_k_best.py` - Модульные тесты

### Компоненты сходства
1. **Текстовое сходство**: Косинусное сходство мультиязычных эмбеддингов MiniLM
2. **Категориальное сходство**: Точное/иерархическое совпадение для региона, способа закупки и т.д.
3. **Числовое сходство**: Гауссово ядро на основе разницы цен
4. **Временное сходство**: Экспоненциальное затухание на основе разницы во времени

### Веса по умолчанию
- Наименование закупки: 0.3
- Описание: 0.2
- Заказчик: 0.15
- Регион: 0.1
- Способ определения поставщика: 0.1
- Цена: 0.1
- Дата: 0.05

## Алгоритм 2: ML-based Approach

### Обзор
Пайплайн неконтролируемого обучения с кластеризацией, контрастивным обучением и индексированием FAISS для поиска сходства в больших масштабах.

### Пайплайн
1. **Предобработка**: Очистка и нормализация данных российских тендеров
2. **Генерация эмбеддингов**: Мультиязычные Sentence Transformers
3. **Снижение размерности**: UMAP до 50-100 измерений
4. **Кластеризация**: HDBSCAN для плотностной кластеризации с обнаружением шума
5. **Обучение сходству**: Контрастивное обучение для уточнения эмбеддингов
6. **Индексирование**: FAISS для эффективного поиска сходства

### Ключевые особенности
- **Неконтролируемое**: Не требует размеченных данных
- **Масштабируемое**: Обрабатывает 100 тыс.+ тендеров
- **Адаптивное**: Изучает специфическое для данных сходство
- **Устойчиво к шуму**: HDBSCAN идентифицирует выбросы

## Сравнительный анализ

| Аспект | Алгоритм 1 | Алгоритм 2 |
|--------|-------------|-------------|
| **Подход** | Правило-взвешенное сходство | Пайплайн неконтролируемого ML |
| **Лучше для** | Наборы данных среднего размера (до 10 тыс.) | Большие наборы данных (100 тыс.+) |
| **Обучение** | Не требуется | Требуется фаза обучения |
| **Производительность** | O(n) линейный поиск | O(log n) с индексированием FAISS |
| **Интерпретируемость** | Высокая (явные веса) | Умеренная (изученное сходство) |
| **Русский язык** | Да (Sentence Transformers) | Да (Sentence Transformers) |

**Рекомендация**: Используйте Алгоритм 1 для быстрого развертывания и интерпретируемости. Используйте Алгоритм 2 для больших наборов данных и адаптивного сходства.

## Скрипт сравнительного анализа

Для практического сравнения производительности алгоритмов 1 и 2 был создан скрипт `compare_algorithms.py`. Этот скрипт позволяет:

1. **Загружать данные** из файла `tenders.json`
2. **Разделять данные** в формате 90-10 (90% для обучения, 10% для тестирования)
3. **Обучать алгоритм 2** на обучающей выборке
4. **Сравнивать производительность** обоих алгоритмов на тестовой выборке
5. **Выводить метрики** сравнения (время выполнения, успешность, оценки сходства)

### Использование скрипта

```bash
python compare_algorithms.py <flags>
```

### Ключевые параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--data` | Путь к файлу tenders.json | `tenders.json` |
| `--max-samples` | Максимальное количество записей для загрузки | Все записи |
| `--max-test-samples` | Максимальное количество тестовых примеров | 5 |
| `--seed` | Seed для воспроизводимости разделения данных | 42 |
| `--output` | Путь для сохранения результатов в JSON | Не сохраняется |
| `--model-path` | Путь к предобученной модели алгоритма 2 (.pkl). Если указан, модель загружается, иначе обучается заново. | Не используется |

### Метрики сравнения

Скрипт вычисляет следующие метрики для каждого алгоритма:

1. **Успешность** - процент успешных выполнений
2. **Среднее время выполнения** - время поиска похожих тендеров
3. **Стандартное отклонение времени** - стабильность производительности
4. **Средняя оценка сходства** - качество найденных совпадений

### Особенности реализации

- **Автоматическое разделение данных** 90-10 с возможностью настройки seed
- **Обработка ошибок** - скрипт продолжает работу даже при сбоях отдельных алгоритмов
- **Оптимизация производительности** - возможность ограничить количество тестовых примеров
- **Сохранение результатов** - результаты можно сохранить в JSON для дальнейшего анализа

### Интерпретация результатов сравнения

При тестировании на реальных данных тендеров наблюдаются значительные различия в производительности алгоритмов:

#### 1. **Разница во времени выполнения (алгоритм 2 быстрее в 200+ раз)**
- **Алгоритм 1** использует линейный поиск O(n) с вычислением составного сходства для каждого кандидата, что требует полного перебора всех исторических тендеров.
- **Алгоритм 2** применяет индексирование FAISS (при включенной опции) или оптимизированный поиск по кластерам, что снижает сложность до O(log n) или O(k), где k ≪ n.

#### 2. **Разница в оценках сходства (алгоритм 2 дает более высокие значения)**
- **Алгоритм 1** вычисляет сходство на основе фиксированных весов и эмбеддингов, не адаптированных к конкретной выборке данных.
- **Алгоритм 2** использует обученные эмбеддинги и метрики, оптимизированные под структуру данных тендеров, что позволяет лучше улавливать семантические связи.

#### 3. **Причины колоссального различия на конкретной выборке**
- **Объем данных**: При 900+ тендерах линейный поиск алгоритма 1 становится затратным (20+ секунд на запрос), в то время как алгоритм 2 использует предварительно построенные индексы.
- **Качество эмбеддингов**: Алгоритм 2 дополнительно обучает similarity learning на данных, что улучшает релевантность результатов.
- **Оптимизация поиска**: Алгоритм 2 фильтрует кандидатов по кластерам, исключая заведомо нерелевантные тендеры.

Эти различия подтверждают архитектурные решения, заложенные в проекте: алгоритм 1 как простое правило-взвешенное решение, алгоритм 2 как масштабируемый ML-пайплайн.

## Прогнозирование цены тендеров

Модуль анализа был расширен двумя подходами для прогнозирования цены новых тендеров на основе исторических данных:

### Вариант 1: SimplePricePredictor (на основе похожих тендеров)

Использует обученный `MLRetrospectiveAnalyzer` для поиска похожих исторических тендеров и вычисляет прогноз цены как взвешенную медиану цен похожих тендеров.

**Основные методы:**
- `median`: медиана цен топ-k похожих тендеров
- `weighted_mean`: среднее взвешенное по оценке сходства
- `weighted_median`: медиана взвешенная по оценке сходства (по умолчанию)
**Использование:**
```python
from price_predictor_simple import SimplePricePredictor
from ml_retrospective import MLRetrospectiveAnalyzer

# Обучить анализатор
analyzer = MLRetrospectiveAnalyzer()
analyzer.train(historical_tenders)

# Создать предиктор
predictor = SimplePricePredictor(analyzer, method='weighted_median')

# Предсказать цену для нового тендера
result = predictor.predict(new_tender, k=5)
print(f"Прогнозируемая цена: {result['predicted_price']:,.2f}")
print(f"Уверенность: {result['confidence']:.2%}")
print(f"Диапазон цен похожих тендеров: {result['price_range'][0]:,.2f} - {result['price_range'][1]:,.2f}")
```

### Вариант 2: RegressionPricePredictor (регрессионная модель)

Использует признаки, извлекаемые `FeatureEngineeringPipeline`, и обучает регрессионную модель (градиентный бустинг) для прогнозирования цены.

**Поддерживаемые модели:**
- `gradient_boosting`: GradientBoostingRegressor (по умолчанию)
- `random_forest`: RandomForestRegressor
- `ridge`: Ridge регрессия
- `lasso`: Lasso регрессия
**Использование:**
```python
from price_predictor_regression import RegressionPricePredictor

# Создать и обучить предиктор
predictor = RegressionPricePredictor(model_type='gradient_boosting')
metrics = predictor.train(training_tenders)

print(f"Обучение завершено. Test RMSE: {metrics['test']['rmse']:,.2f}")
print(f"Test R²: {metrics['test']['r2']:.3f}")

# Предсказать цену
result = predictor.predict(new_tender)
print(f"Прогнозируемая цена: {result['predicted_price']:,.2f}")
print(f"Уверенность: {result['confidence']:.2%}")

# Сохранить модель
predictor.save('price_regression_model.joblib')
```

### Сравнение точности методов

Для сравнения точности обоих подходов используйте скрипт `compare_price_predictors.py`:

```bash
python compare_price_predictors.py --data tenders.json --max-samples 1000
```

1. Загрузку и разделение данных (60% train, 20% validation, 20% test)
2. Обучение обоих предикторов

   - MAE (Mean Absolute Error)
   - RMSE (Root Mean Square Error)
   - MAPE (Mean Absolute Percentage Error)
   - R² (Coefficient of Determination)
4. Создание визуализаций сравнения
5. Сохранение результатов в JSON
